| 항목 | 결정 | 비고 |

## 3. 비용 영향 요소

### 비용 절감 포인트

| 항목 | 절감 내용 | 영향도 |
|------|----------|--------|
| **GGPass SSO** | 로그인/가입 개발 불필요 | 높음 |
| **빌링 시스템** | 내부 결제팀 API 활용 | 높음 |
| **스케줄 API** | 내부 DB 연동, 별도 구축 불필요 | 중간 |
| **디자인** | BX팀 내부 처리 가능성 | 중간 |
| **사이트 범위 축소** | Watch + Collections 2개 메뉴만 | 높음 |

### 비용 증가 요소

| 항목 | 내용 | 영향도 |
|------|------|--------|
| **멀티뷰 트래픽** | 4분할 시 트래픽 비용 4배 | 높음 |
| **커스텀 플레이어** | 포커 특화 기능 개발 필요 | 높음 |
| **TV 앱** | 플랫폼별 개발 필요 | 중간 |

## 5. 권고 사항

1. **컨설팅 단계 선행 권장**

2. **디자인 역할 명확화 필요**
   - 내부 BX팀 vs 외부 개발사 역할 분담 확정 필요

3. **제휴 검토**
   - ODK 박스 (온디맨드 코리아) 연계로 북미 호텔 LGTV 진출 가능성

# Part B. 상세 기술 분석 (기술팀용)

## 1. 스트리밍 기술

### 1.1 지연 시간 (Latency)

| 옵션 | 지연 시간 | 비용 | 결정 |
|------|----------|------|------|
| 초저지연 (LLHLS) | 2-5초 | 높음 | **제외** |
| 표준 HLS | 15-30초 | 중간 | **채택** |
| 의도적 지연 | 30분~1시간 | 낮음 | **운영 정책** |

**결정 근거**: 포커 방송은 핸드(카드) 노출로 인한 스포일러 방지를 위해 30분~1시간 지연 방송 운영. 따라서 기술적 초저지연은 불필요.

### 1.2 어댑티브 스트리밍

| 프로파일 | 비트레이트 | 대상 |
|----------|-----------|------|
| 1 | 최저 (음성만) | 극저속 네트워크 (미국 텍사스 등) |
| 2 | 360p | 동남아 등 저속 네트워크 |
| 3 | 480p | 모바일 3G |
| 4 | 720p | 모바일 4G/5G |
| 5 | 1080p | PC/TV |
| 6 | 1080p+ | 고속 네트워크 |

**권장**: 6개 프로파일 (글로벌 스탠다드)

### 1.3 4K 지원

| 항목 | 현재 상태 | 필요 조치 |
|------|----------|----------|
| 카메라 | 4K 미지원 | 전면 교체 |
| 케이블 | 기존 규격 | 전면 교체 |
| 인코딩 인프라 | 1080p 기준 | 업그레이드 |
| 트래픽 비용 | - | 약 4배 증가 |

**결정**: 1차 출시에서 제외, 프리미엄 구독 모델 2차 검토

## 2. 멀티뷰/PLP 구현

### 2.1 멀티뷰 구조

```
┌─────────────────────────────────────┐
│           메인 화면 (PGM)            │
│         (1080p 풀 스트림)            │
├─────────┬─────────┬─────────┬───────┤
│ 카메라1  │ 카메라2  │ 카메라3  │ 카메라4│
│(썸네일)  │(썸네일)  │(썸네일)  │(썸네일)│
└─────────┴─────────┴─────────┴───────┘
```

### 2.2 기술 옵션

| 방식 | 트래픽 | 품질 | 권장 |
|------|--------|------|------|
| 풀 스트림 4개 | 4x | 최고 | 비권장 (비용) |
| 메인 풀 + 서브 저화질 | 1.5x | 좋음 | 권장 |
| 라이브 썸네일 (30초 갱신) | 1.1x | 보통 | **채택** |
| 스튜디오 믹싱 (단일 출력) | 1x | PD 의존 | 일부 채택 |

### 2.3 카메라 구성 (피처 테이블)

| 카메라 | 대상 | 수량 |
|--------|------|------|
| 플레이어 캠 | 개별 플레이어 | 9대 |
| 보드 캠 | 커뮤니티 카드 | 1대 |
| **합계** | | **10대** |

### 2.4 플랫폼별 지원

| 플랫폼 | 멀티뷰 | 카메라 선택 | 비고 |
|--------|--------|------------|------|
| Web | O | O | 전체 기능 |
| iOS | O | O | 전체 기능 |

## 3. 자막/다국어

### 3.1 자막 처리 흐름

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  영상    │───▶│ AI 추출  │───▶│ 편집기   │───▶│ 번역     │
│ (음성)   │    │ (STT)    │    │ (수정)   │    │ (20개국) │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
```

### 3.2 언어 지원

| 구분 | 언어 수 | 비고 |
|------|--------|------|
| 기술적 가능 | 70개 | AWS 번역기 스펙 |
| 1차 계획 | 20개 | 주요 시장 |
| 원본 언어 | 영어 | 번역 품질 최적화 |

### 3.3 실시간 자막 제한

- **완전 자동화 어려움**: 딕션/발음 품질에 따라 정확도 차이
- **권장 방식**: VOD에서 AI 추출 → 편집 → 번역
- **라이브**: 인력 투입 필요 (자동화 미권장)

## 4. DRM/보안

### 4.1 Multi-DRM 구성

| DRM | 플랫폼 | 필수 여부 |
|-----|--------|----------|
| Widevine | Chrome, Android | 필수 |
| FairPlay | Safari, iOS, Apple TV | 필수 |
| PlayReady | Edge, Windows, Xbox | 필수 |

### 4.2 브라우저 제약

| 브라우저 | DRM | 대응 |
|----------|-----|------|
| Chrome | Widevine | 정상 작동 |
| Safari | FairPlay | 정상 작동 |
| Edge | PlayReady | 정상 작동 |
| Chrome (일부) | FairPlay | **미지원** → Edge 안내 |

### 4.3 지오블로킹

| 방식 | 정확도 | 권장 |
|------|--------|------|
| IP 기반 국가 차단 | 95%+ | 채택 |
| VPN 우회 방지 | 80-90% | 부분 채택 |
| 완전 차단 | 불가능 | - |

**정책 옵션**:
1. 사이트 단위 차단 (접속 자체 불가)
2. 영상 단위 차단 (사이트 접근 가능, 재생만 불가)

# Part D. 미팅 회의록 요약 (부록)

## 미팅 정보

| 항목 | 내용 |
|------|------|
| **업체** | 맑음 소프트 (미킨디오/Mega Cloud) |
| **형식** | 기술 검증 미팅 |
| **시간** | 약 63분 |
| **참석자** | 업체 담당자 1명 + GGPoker/WSOPTV 측 4명 |

## 주요 논의 타임라인

### 1. 서비스 개요 확인 (00:00~05:00)
- 글로벌 서비스, 동시접속 50만명 목표 확인
- 초저지연 불필요 확정 (30분~1시간 지연 방송, 스포일러 방지)
- 어댑티브 스트리밍 6개 프로파일 적용

### 2. 자막/다국어 (04:00~05:00)
- 라이브 실시간 자막 희망 → 사람 손 필요 (완전 자동화 어려움)
- VOD 자막: AI 추출 후 편집 → 20개국 언어 번역 가능
- 영어 기반 원본 → 타 언어 번역 품질 우수

### 3. DRM/보안 (05:00~07:00)
- 멀티DRM (Widevine, FairPlay, PlayReady) 적용 권장
- 브라우저 제약 있음 (Chrome에서 특정 DRM 제한)
- 지오블로킹: IP 기반, VPN 완전 차단은 불가

### 4. 멀티뷰/PLP (07:00~10:00)
- NBA처럼 4개 화면 동시 표시 가능
- 4개 이상 시 기기 사양 문제 발생
- PIP(작은 화면)도 가능하나 트래픽 비용 4배 증가
- 스튜디오에서 믹싱하면 트래픽 1개로 절감 가능

### 5. 방송 환경/카메라 (28:00~32:00)
- 피처 테이블: 9-10대 카메라 (플레이어당 1대 + 보드캠)
- PLP(Picture-Layout-Picture) 구현 가능
- 라이브 썸네일: 30초마다 갱신 (트래픽 절약)
- TV 앱에서는 PD 연출 화면(PGM)만 제공 권장

### 6. AI 기능 (20:00~28:00)
- 자막 기반 챕터/요약: 가능 (LLM 기반)
- VLM(비디오 분석): 비용 높음, 별도 업체 필요
- 하이라이트 자동 추출: 프로덕션팀에서 이미 수작업 진행 중

### 7. 4K 지원 논의 (35:00~38:00)
- 프리미엄 구독 모델에서 4K 검토
- 현실적 문제: 카메라/케이블/인프라 전면 교체 필요
- 프로덕션 비용 감당 불가 수준
- 1차에서는 1080p로 확정

### 8. 프로젝트 범위 확정 (54:00~60:00)
- WSOPTV.com 별도 사이트 생성
- 기존 WSOP.com과 링크 연결
- 메뉴: Watch(라이브) + Collections(VOD) 위주
- 뉴스, 전적, 플레이어 정보: 제외 (외부 링크)

### 9. 역할 분담 (51:00~63:00)
- **GGPoker**: GGPass SSO, 빌링 API, 스케줄 API, 디자인(BX팀)
