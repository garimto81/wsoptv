## 질문 목록 (14개)

### 플레이어/SDK (4개)

#### Q1. WAVVE에서 사용한 플레이어 기술 스택은? 자체 개발 vs 외부 OVP 기반?

| 항목 | 내용 |
|------|------|
| **왜 중요한가** | WAVVE는 한국 최대 OTT 서비스로 4.2M MAU를 처리합니다. 이 규모를 안정적으로 운영한 플레이어 기술이 무엇인지 파악해야 WSOPTV에 적용 가능한지 판단할 수 있습니다. |
| **기술 배경** | 플레이어는 자체 개발(완전한 커스터마이징 가능, 유지보수 부담 큼) 또는 외부 OVP 기반(빠른 개발, 라이선스 비용 발생)으로 구축할 수 있습니다. |
| **답변** | ☐ |

---

#### Q2. 멀티뷰(다중 카메라) 동시 재생 구현 경험 및 방식은?

| 항목 | 내용 |
|------|------|
| **왜 중요한가** | WSOPTV의 핵심 기능인 멀티뷰는 메인 테이블 영상과 개별 플레이어 카메라를 동시에 표시해야 합니다. WAVVE가 스포츠 중계 등에서 멀티뷰를 구현한 경험이 있다면 WSOPTV에 적용 가능합니다. |
| **기술 배경** | 멀티뷰는 여러 스트림을 동시에 디코딩하고 렌더링해야 합니다. 대역폭 최적화(저화질 썸네일 스트림 사용)와 스트림 동기화(모든 카메라가 같은 시점을 보여줌)가 기술적 도전입니다. |
| **답변** | ☐ |

---

#### Q3. Quick VOD (Live→VOD 실시간 전환) 기술 세부사항은?

| 항목 | 내용 |
|------|------|
| **왜 중요한가** | Quick VOD는 WAVVE의 차별화 기능으로, 라이브 방송 중에도 이미 방송된 부분을 VOD처럼 되감기/빨리감기할 수 있습니다. |
| **기술 배경** | 일반적으로 라이브 스트림은 실시간만 시청 가능하고, VOD는 방송 종료 후 별도 처리가 필요합니다. Quick VOD는 라이브 스트림을 실시간으로 세그먼트 단위로 저장하여 즉시 VOD로 제공하는 기술입니다. |
| **답변** | ☐ |

---

#### Q4. 자막/다국어 지원 아키텍처는? 실시간 자막 가능한가요?

| 항목 | 내용 |
|------|------|
| **왜 중요한가** | WSOPTV는 글로벌 서비스로 20개 언어 자막을 지원해야 합니다 (GG POKER 지원 언어 기준). WAVVE는 주로 한국어 서비스이므로, 다국어 지원 경험과 실시간 자막 기술력을 확인해야 합니다. |
| **기술 배경** | 자막은 사전 제작(VOD에 미리 자막 파일 첨부)과 실시간 생성(라이브 중 음성을 텍스트로 변환)으로 나뉩니다. |
| **답변** | ☐ |

### OTT/앱 (4개)

#### Q5. WAVVE 앱 개발 시 사용한 기술 스택은? (Web/iOS/Android/TV)

| 항목 | 내용 |
|------|------|

#### Q6. TV 앱 개발 경험 범위는? Samsung, LG 외에 Roku/Fire TV 가능한가요?

| 항목 | 내용 |
|------|------|
| **왜 중요한가** | WSOPTV는 글로벌 서비스이므로 북미 시장에서 인기 있는 Roku, Fire TV도 필수입니다. WAVVE는 한국 시장 중심이라 Samsung/LG 경험은 있지만, 글로벌 TV 플랫폼 경험이 제한적일 수 있습니다. |
| **기술 배경** | Samsung Tizen, LG webOS는 웹 기술 기반으로 개발 방식이 유사합니다. Roku는 BrightScript라는 독자 언어를, Fire TV는 Android 기반입니다. |
| **답변** | ☐ |

#### Q7. 글로벌 서비스 확장 경험은? KOCOWA 통합, wavve go 해외 서비스 등

| 항목 | 내용 |
|------|------|

| **기술 배경** | 글로벌 서비스는 다국어 앱 관리, 글로벌 인프라 구성, 현지화가 필요합니다. |
| **답변** | ☐ |

#### Q8. 앱 성능 최적화 경험 (4.2M MAU 처리)은?

| 항목 | 내용 |
|------|------|
| **왜 중요한가** | WAVVE의 4.2M MAU는 한국 OTT 2위 규모입니다. 이 트래픽을 안정적으로 처리한 경험은 WSOPTV의 50만 동시접속 목표를 달성하는 데 중요한 레퍼런스입니다. |
| **기술 배경** | 대규모 트래픽 처리에는 캐싱 전략, 로드밸런싱, 오토스케일링, 데이터베이스 최적화 등이 필요합니다. |
| **답변** | ☐ |

### 인프라 (4개)

#### Q9. CDN 파트너 및 글로벌 커버리지는?

| 항목 | 내용 |
|------|------|
| **왜 중요한가** | WAVVE는 한국 시장 중심이므로 한국 CDN(SKT, KT 등)을 주로 사용할 가능성이 높습니다. WSOPTV는 북미, 유럽, 아시아 전역에서 서비스해야 하므로 글로벌 CDN 커버리지가 필수입니다. |
| **기술 배경** | 한국 로컬 CDN은 한국 내에서는 빠르지만, 해외 PoP가 부족합니다. 글로벌 CDN(Akamai, CloudFront 등)은 전 세계 PoP를 보유합니다. |
| **답변** | ☐ |

---

#### Q10. 대규모 동시접속 처리 아키텍처는?

| 항목 | 내용 |
|------|------|
| **왜 중요한가** | WSOPTV는 WSOP Main Event 기간에 50만 동시접속을 목표로 합니다. WAVVE가 대규모 동시접속을 처리한 아키텍처와 경험을 확인해야 합니다. |
| **기술 배경** | 대규모 동시접속 처리에는 마이크로서비스 아키텍처, 오토스케일링, 로드밸런싱, 캐싱(Redis 등), 메시지 큐(Kafka 등)가 필요합니다. |
| **답변** | ☐ |

#### Q11. 해외 지역 서비스 시 인프라 구성 방안은?

| 항목 | 내용 |
|------|------|
| **왜 중요한가** | WSOPTV는 글로벌 서비스이므로 인프라도 글로벌하게 구성해야 합니다. 한국에만 서버가 있으면 북미/유럽 사용자는 높은 지연 시간으로 불편을 겪습니다. |
| **기술 배경** | 멀티 리전 배포는 여러 지역(예: 서울, 버지니아, 프랑크푸르트)에 서버를 배치하여 사용자와 가까운 서버에서 서비스를 제공합니다. |
| **답변** | ☐ |

#### Q12. 외부 시스템(빌링/CMS/SSO) 연동 API 설계 경험은?

|------|------|
| **왜 중요한가** | WSOPTV는 GGPoker의 빌링 시스템, 스케줄 데이터, GGPass SSO와 연동해야 합니다. 외부 시스템 연동 경험이 풍부해야 원활한 통합이 가능합니다. |
