## 1. 프로젝트 개요

### 1.1 배경
GGPoker는 전세계 1위 온라인 포커 업체로서, WSOP(World Series of Poker) 인수 후 글로벌 포커 팬들을 위한 프리미엄 OTT 스트리밍 서비스 **WSOPTV**를 구축하고자 합니다.

### 1.2 서비스 비전
- **서비스명**: WSOP TV
- **대상**: 전세계 포커 팬 (MAU 100만명 목표)
- **핵심 기능**: Live 토너먼트 스트리밍 + VOD 아카이브
- **목표 동시접속**: 50만명 (WSOP Main Event 기간)

### 1.3 플랫폼
- Web (PC/Mobile 반응형)
- iOS App
- Android App
- TV App (Samsung, LG, Roku, Fire TV, Apple TV)

## 4. 레퍼런스 모델: NBA League Pass

### 4.1 개요

NBA League Pass는 WSOPTV가 벤치마킹해야 할 핵심 레퍼런스 모델입니다. NBA Digital(NBA와 TNT Sports 합작)이 운영하며, Microsoft Azure 기반의 "Next Gen 통합 디지털 플랫폼"으로 구동됩니다.

### 4.2 핵심 기능 분석

#### 멀티뷰 (Multiview)

| 기능 | 상세 |
|------|------|
| **동시 시청** | 최대 4개 게임 동시 시청 가능 |
| **커스터마이징** | 메인 화면(대형) + 서브 화면(소형) 자유 구성 |
| **디바이스 지원** | Connected TV, 모바일, 웹 전체 지원 |
| **Prime Video 연동** | Amazon Prime Video와 통합 멀티뷰 (NBA on Prime + League Pass) |

**WSOPTV 적용점**: 포커 테이블 멀티뷰 (메인 테이블 + 개별 플레이어 카메라)

#### 라이브 스트리밍

| 기능 | 상세 |
|------|------|
| **게임 수** | 시즌당 800+ 라이브 경기 |
| **중계 옵션** | 홈/어웨이 중계, 다국어 중계 (스페인어, 포르투갈어, 한국어) |
| **특별 중계** | NBA Strategy Stream, NBA HooperVision (대체 중계) |
| **카메라 앵글** | 사이드라인 캠, 백보드 캠, 플레이어 캠 |
| **In-Arena Stream** | 하프타임/경기 휴식 중 경기장 내 영상 (Premium) |

**WSOPTV 적용점**: 멀티 카메라 앵글 (테이블 전체, 개별 플레이어, 보드 카메라)

#### VOD/아카이브

| 기능 | 상세 |
|------|------|
| **아카이브 범위** | 2012-13 시즌부터 현재까지 전체 게임 |
| **요약 형식** | 다양한 Condensed Game 포맷 제공 |
| **다운로드** | 오프라인 시청용 다운로드 지원 |
| **스마트 리와인드** | AI 기반 주요 장면 되감기 |

**WSOPTV 적용점**: 토너먼트 전체 아카이브, 핸드별 하이라이트 클립

#### AI 기반 기능

| 기능 | 상세 |
|------|------|
| **NBA Insights** | AI 기반 실시간 텍스트 업데이트 (박스 스코어 이상의 내러티브) |

| **콘텐츠 로컬라이제이션** | Generative AI로 게임 리캡 다국어 자동 생성 (프랑스어, 포르투갈어, 스페인어) |

**WSOPTV 적용점**: AI 핸드 분석, 자동 하이라이트 감지, 다국어 자막 자동 생성

## 부록

### A. 스트리밍 기술 상세 분석 (미팅 기반)

#### A.1 지연 시간 (Latency) 옵션

| 옵션 | 지연 시간 | 비용 | WSOPTV 결정 |
|------|----------|------|-------------|
| 초저지연 (LLHLS) | 2-5초 | 높음 | **제외** |
| 표준 HLS | 15-30초 | 중간 | **채택** |
| 의도적 지연 | 30분~1시간 | 낮음 | **운영 정책** |

**결정 근거**: 포커 방송은 핸드(카드) 노출로 인한 스포일러 방지를 위해 30분~1시간 지연 방송 운영. 따라서 기술적 초저지연은 불필요.

#### A.2 어댑티브 스트리밍 프로파일 (권장)

| 프로파일 | 비트레이트 | 대상 네트워크 |
|----------|-----------|---------------|
| 1 | 최저 (음성만) | 극저속 (미국 텍사스 등) |
| 2 | 360p | 동남아 등 저속 |
| 3 | 480p | 모바일 3G |
| 4 | 720p | 모바일 4G/5G |
| 5 | 1080p | PC/TV |
| 6 | 1080p+ | 고속 네트워크 |

**권장**: 6개 프로파일 (글로벌 스탠다드)

#### A.3 멀티뷰 구현 방식 비교

| 방식 | 트래픽 배수 | 품질 | WSOPTV 결정 |
|------|------------|------|-------------|
| 풀 스트림 4개 | 4x | 최고 | 비권장 (비용) |
| 메인 풀 + 서브 저화질 | 1.5x | 좋음 | 권장 |
| 라이브 썸네일 (30초 갱신) | 1.1x | 보통 | **채택** |
| 스튜디오 믹싱 (단일 출력) | 1x | PD 의존 | 일부 채택 |

**권장**: 라이브 썸네일 방식 채택 (트래픽 최적화)
